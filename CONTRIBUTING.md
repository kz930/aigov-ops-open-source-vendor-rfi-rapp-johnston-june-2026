# Contributing to Safe-RFI

Thanks for helping improve Safe-RFI.

## How to contribute

1. Open an issue describing the problem or proposal.
2. Keep changes focused and scoped to the issue.
3. Preserve static-site portability (no server-side dependencies).
4. Ensure copy and UI updates remain neutral, evidence-oriented, and accessible.
5. Submit a pull request with clear rationale.

## Quality expectations

- Keep the project GitHub Pages ready.
- Maintain keyboard accessibility and mobile-friendly layout.
- Avoid introducing external dependencies unless clearly justified.
- Keep links valid and documentation coherent.

## Development notes

Safe-RFI is implemented as static HTML/CSS/JS.

- Root `index.html` is the repository landing page.
- `webapp/index.html` is the primary application entry.
- `webapp/vendor-rfi-advisor.html` is retained for compatibility.
